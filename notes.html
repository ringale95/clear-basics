Fundamentals:

  • When you visit a website, your browser (client) sends an HTTP request to the server & receives an HTTP response which includes an HTML document that represents the page you're visiting. The browser reads the document & constructs a DOM (Document Object Model). Once the DOM is ready, the browser renders the target page.
  • To view HTTP requests we can use the Network Tab.
  • DOCTYPE declaration specifies the version of HTML being used.
  • The root element in HTML is the html element.
  • Styles types: Embedded, external, and inline.
  • External stylesheets: styles.css. Link the stylesheet document: <link rel="stylesheet" href="styles.css">.
  • Override the external stylesheet by defining the rules in the same HTML.
  • Inline styles.

Normalize CSS

  • Normalize.css will define CSS for the complete page.
  • Selectors: Type, id, class, attribute.
  • ID: <section id="products"></section>.
  • <a href="http://www.google.com" target="_blank">Google</a>.
    • To access all 'a' tags with target attribute: a[target="_blank"] { color: orange; }.
    • To access all 'a' tags with href containing "google": a[href*="google"] { color: orange; } (Use this instead of typing the actual URL as it is very fragile).
    • To check if URL starts with a given word: a[href^="http"] { color: orange; }.
    • To check if URL ends with a given word: a[href$="http"] { color: orange; }.

Relational selectors

Select using p which includes all 'p':

#products p { color: orange; }

Select only the first child of products:

#products > p { color: orange; }

Select the immediate sibling of products:

#products + p { color: orange; }

Select all siblings of products:

#products ~ p { color: orange; }

Pseudo class selectors

Types of pseudo selectors are: It starts with :

  • article :first-child {}
  • article p:first-child {}
  • article :first-of-type {}
  • article p:first-of-type {}
  • article p:last-child {}
  • article p:last-of-type {}
  • ul li:nth-child(odd) {}
  • a :visited, a :link {}
  • a :hover, a :focus {}

Pseudo elements selectors

Starts with ::

  • p ::first-letter {}
  • p ::first-line {}
  • p ::selection {}
  • p ::before {}

Basics:

For CSS property: box-sizing: border-box; the size of content is only shown or calculated and paddings and borders aren't included. Therefore, apply it universally. But it is not applied to pseudo elements so specifically mention it like:

*, *::before, *::after { box-sizing: border-box; }

overflow: hidden -> will hide extra content and show scroll

overflow: auto -> scrollbar only shows if more content

Units:

  • Absolute units:
    • px, pt, in, cm, mm
  • Relative units:
    • % -> relative to the size of parent. By default, the block-level elements' height is 0 and width = 100%. Height cannot be used.
    • vw -> size of the viewport. We can size as per height as well and width too = 100vh and width = 50vw
    • vh -> size of the viewport
    • em, rem -> font-size = em = width = 10em means 10 * 16px (font-size default) and rem is root level which is 16px but em is whatever is parent font-size. The benefit is properly resizable and adjusts as per font size of HTML defined in windows settings.

VH height can cause some issues on mobile screens. So use media queries.

Positioning:

  • By default, all boxes' positions of all elements are "Static" meaning they are not positioned. To position these elements change the position property to relative where we can position it relative to the original position.
  • position: relative -> we can position it relative to its original position by left: -4rem; or right: 4rem;.
  • z-index property is the depth. By default, it's 0 for all elements. If 1 means closer to us and -1 means away.
  • Absolute: Here we are removing the element from its original position & positioning it. Positioned relative to its nearest ancestor.

Responsive Web:

  • View port: We can use the viewport meta tag to define the viewport dimensions.
  • Media queries:
    • min-width -> applies this rule if the width of the device is >= 37.5em
    • max-width -> applies this rule if the width of the device is < 37.5em

Flexbox:

  • New CSS layout mode. Will arrange elements inside the flex container relative to each other. Works with parent-child relationships only. Useful for 1D layout (rows and columns) & the default is a row.
  • Flex container: display: flex; to make an element a flex container.
  • Flex item: Children of the flex container. When flex-direction is a row, the main axis is horizontal & the cross-axis is vertical. When flex-direction is a column, the main axis is vertical & the cross-axis is horizontal.
  • Align items:
    • justify-content: along main axis
    • align-items: along cross axis
  • flex-wrap: wrap; Items that don't fit in one line will be wrapped and pushed down maintaining the size
  • align-content: center; This align-content will adjust the complete content of flexbox as per main axis

Sizing items in flex:

Not to the container but to items:

  • flex-basis: (will override the normal width or height depending on flex-direction: row or column)
  • flex-grow: (growth factor of flex items. Grow in the same factor)
  • flex-shrink: (how the elements should shrink)
  • flex: (grow, shrink, basis)

Grids:

Define a grid using container:

  • display: grid
  • grid-template-rows: (define the number of rows)
  • grid-template-columns: (define the number of columns)
  • justify-items: along horizontal axis
  • align-items: along vertical axis
